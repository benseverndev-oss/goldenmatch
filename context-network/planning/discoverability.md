# Discoverability — the 2026-06-13 public-surface audit arc

An accuracy-first, value-forward sweep of every surface a human or an AI crawler
hits before installing: the root README, the `llms.txt` family, the GitHub repo
About, package-registry metadata, and the external discovery channels. Shipped on
PR #883 (docs-only) plus live GitHub-settings edits. Sibling to the
[surface-hardening](surface-hardening.md) arc (which is the CLI/TUI/HTTP/API
*functional* surfaces — distinct from these *discovery* surfaces).

## The governing rule
**Lead value-forward, but every public number is verified against source before it
ships.** The audit pass for each surface was: (1) diff what the repo actually offers
against what the surface claims, (2) fix every inaccuracy, (3) restructure to surface
the under-sold high-value work. Numbers are checked against the bake-off doc,
`Cargo.toml` deps, `__all__`, MCP `Tool(` / A2A `_SKILLS` counts, registry metadata —
never asserted from memory.
