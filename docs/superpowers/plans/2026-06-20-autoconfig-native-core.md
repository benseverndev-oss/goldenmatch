# Auto-Config Native Core Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port goldenmatch's deterministic auto-config decision logic (the 8-rule execution planner + the column classifier) into a pyo3-free Rust crate `goldenmatch-autoconfig-core`, bound to Python (`goldenmatch-native`) and WASM/TS, with a Python-oracle golden-vector parity harness run by every surface in CI.

**Architecture:** A new standalone `-core` crate (mirrors `fingerprint-core`/`score-core`) holds two pure layers — Layer 1 `decide_plan(PlannerInput) -> ExecutionPlan` and Layer 2 `classify_columns(&[ColumnStats]) -> Vec<ColumnProfile>`. The planner's live env/import probes become an explicit `Capabilities` input. Path A (stats-in, default) is WASM-trivial; path B (arrow-in, opt-in `feature="arrow"`) is the bigger-samples enabler. Thin `#[pyfunction]` shims in `goldenmatch-native` + a `wasm-bindgen` wrapper bind it; pure-Python/TS stay as the parity oracle and the fallback. Default-off, validate-then-cutover.

**Tech Stack:** Rust (`fancy-regex`, `serde`, optional `arrow`), PyO3 abi3 (`goldenmatch-native`), `wasm-bindgen`/`wasm-pack`, Python 3.11+ (polars), TypeScript (tsup + vitest).

---

## Pre-flight: decisions locked before any task

1. **Spec is the porting oracle.** `docs/superpowers/specs/2026-06-20-autoconfig-native-core-design.md` quotes the exact Python source (patterns, `_classify_by_name`/`_classify_by_data`/`_is_year`, `_guess_type`, the 8 planner rules + `auto_chunk_size`, the merge precedence at lines 350-368). Port line-by-line against it. The golden vectors are the completeness gate.

2. **Gate mechanism — reuse the existing one, no new env var.** The codebase already gates native per-component via `native_enabled(component)` + the `_GATED_ON` frozenset in `core/_native_loader.py`. Auto-config uses component name `"autoconfig"`. It ships **NOT in `_GATED_ON`** (so `GOLDENMATCH_NATIVE=auto`/unset → pure Python). The parity CI lane forces it on with `GOLDENMATCH_NATIVE=1`. Cutover = add `"autoconfig"` to `_GATED_ON` (one line) once parity is green on all surfaces. This is the exact path `clustering`/`block_scoring` took. (Deviation from the spec's `GOLDENMATCH_AUTOCONFIG_NATIVE`: a parallel env var duplicates the gate — YAGNI. Documented here intentionally.)

3. **Parity oracle generation.** A Python script (`scripts/gen_autoconfig_golden.py`) imports the *pure-Python* `apply_planner_rules` and `profile_columns` and emits two JSON vector files into a shared location both Rust and TS read: `packages/rust/extensions/autoconfig-core/golden/planner_vectors.json` and `classifier_vectors.json`. Each vector is `{input, expected}`. Regenerating is deterministic (no clocks/RNG in these paths). The TS parity test reads the *same* JSON (copied/symlinked into `packages/typescript/goldenmatch/tests/parity/fixtures/` via the existing emitter pattern).

4. **Staging.** Stages A→D land first (core crate + Python binding + harness + bench). Stage E (WASM/TS) lands second. Each stage is independently committable and green.

5. **Version bumps.** Adding symbols to `goldenmatch-native` requires bumping BOTH `packages/rust/extensions/native/Cargo.toml` `[package].version` AND `packages/rust/extensions/native/pyproject.toml` `[project].version` in lockstep (currently `0.1.5` → `0.1.6`). Republish discipline is a release concern (out of this plan), but the bump lands here.

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "kebab-case")]
pub enum BackendName { PolarsDirect, Chunked, Duckdb, Ray, Bucket }

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum ClusteringStrategy { InMemory, PartitionedUnionFind, StreamingCc }

// NOTE: NO `None` variant. Python's pair_spill_threshold is `Literal[...] | None`
// and 5 of 8 rules (pathological/simple/fast_box/bucket_suggested/user_override)
// leave it None -> JSON `null`. Model the absence as Option (below), NOT as an
// enum variant: a `None` variant under snake_case would serialize to the string
// "none" and fail to round-trip the oracle's `null`. This is the load-bearing
// serde detail Task C3 re-verifies.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum SpillThreshold { Ram, Duckdb, DiskPerWorker }

#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
pub struct RuntimeProfile { pub available_ram_gb: f64, pub cpu_count: u32, pub disk_free_gb: f64 }

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Capabilities {
    pub bucket_available: bool,
    pub ray_available: bool,
    pub ray_auto_select: bool,
    pub user_backend: Option<BackendName>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PlannerInput {
    pub n_rows_full: u64,
    pub estimated_pair_count: u64,
    pub runtime: RuntimeProfile,
    pub caps: Capabilities,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct ExecutionPlan {
    pub backend: BackendName,
    pub chunk_size: Option<u64>,           // Python None -> null (simple/fast_box/etc.)
    pub max_workers: u32,
    pub pair_spill_threshold: Option<SpillThreshold>,  // Python None -> null (5 of 8 rules)
    pub clustering_strategy: ClusteringStrategy,
    pub rule_name: String,                 // ALWAYS the fully-resolved post-dispatch name
}

// Named constants — keep identical to autoconfig_planner_rules.py.
const SIMPLE_PLAN_MAX_ROWS: u64 = 100_000;
const SIMPLE_PLAN_MAX_PAIRS: u64 = 50_000_000;
const FAST_BOX_MIN_RAM_GB: f64 = 32.0;
const BUCKET_SUGGESTED_MAX_ROWS: u64 = 750_000;
const PAIR_SCORE_BYTES: u64 = 64;
const BUCKET_RAM_SAFETY_FRACTION: f64 = 0.5;
const CHUNKED_MAX_PAIRS: u64 = 5_000_000_000;
const CHUNKED_MIN_RAM_GB: f64 = 16.0;
const CHUNKED_TARGET_RAM_USE_FRACTION: f64 = 0.6;
const CHUNKED_BYTES_PER_ROW: u64 = 1024;
const RAY_MIN_ROWS: u64 = 50_000_000;
const DUCKDB_MIN_PAIRS: u64 = 5_000_000_000;
const DUCKDB_MAX_RAM_GB: f64 = 16.0;
const DUCKDB_MAX_WORKERS: u32 = 8;

pub fn auto_chunk_size(n_rows_full: u64, available_ram_gb: f64) -> u64 {
    let estimated_gb = (n_rows_full * CHUNKED_BYTES_PER_ROW) as f64 / (1024f64.powi(3));
    let denom = (available_ram_gb * CHUNKED_TARGET_RAM_USE_FRACTION).max(0.001);
    let target_chunks = (estimated_gb / denom).ceil().max(1.0) as u64;
    let chunk = n_rows_full / target_chunks;          // integer floor division (matches Python //)
    chunk.clamp(10_000, 1_000_000)
}
```

NOTE the float→int parity detail: Python uses `math.ceil` then integer `//`. Keep `.ceil()` then integer division. The golden vectors (Task C5) guard the boundaries.

```rust
//! Thin PyO3 shims over goldenmatch-autoconfig-core (JSON in / JSON out).
use pyo3::prelude::*;
use pyo3::exceptions::PyValueError;
use goldenmatch_autoconfig_core::{decide_plan, classify_columns, PlannerInput, ColumnStats};

#[pyfunction]
pub fn autoconfig_decide_plan(input_json: &str) -> PyResult<String> {
    let input: PlannerInput = serde_json::from_str(input_json)
        .map_err(|e| PyValueError::new_err(format!("bad PlannerInput json: {e}")))?;
    let plan = decide_plan(&input);
    serde_json::to_string(&plan).map_err(|e| PyValueError::new_err(e.to_string()))
}

#[pyfunction]
pub fn autoconfig_classify_columns(cols_json: &str) -> PyResult<String> {
    let cols: Vec<ColumnStats> = serde_json::from_str(cols_json)
        .map_err(|e| PyValueError::new_err(format!("bad ColumnStats json: {e}")))?;
    let out = classify_columns(&cols);
    serde_json::to_string(&out).map_err(|e| PyValueError::new_err(e.to_string()))
}
```
