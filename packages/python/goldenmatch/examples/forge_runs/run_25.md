# Algorithm Forge — run to 25 accepted proposals

Generated by `scripts/algorithm_forge.py` (driven by Claude Opus 4.8). The harness
loops survey → structurize → propose → validate over the history of string-similarity
/ distance algorithms and stops once a target number of proposals clear an adversarial
`yes` verdict. This run targeted **25 `yes`** for entity-resolution string similarity.

**Result:** 25 `yes` after 31 proposals evaluated (4 `maybe`, 2 `failed` discarded en
route — e.g. `PhonEdit` ❌ = rebadged Editex 1996; "just use a transformer" ❌ is not an
implementable mechanism). Scores are 0–100 per axis, kept honest and spread.

## Gaps that drove the proposals

- **G1 — One-to-many abbreviation/acronym alignment.** Every hybrid (Monge–Elkan,
  Soft-TFIDF) assumes a token↔token bijection, but `IBM` ↔ `International Business
  Machines` is one token aligning to a *span*. No standard similarity models this.
- **G2 — Corpus weighting is siloed from the error model.** IDF (which expansions are
  rare/informative) is never combined with a directional, generative token model.
- **G3 — Scores aren't calibrated probabilities**, so thresholds are per-dataset magic
  numbers.

## Headline finding

The real headroom in ER string similarity is not another edit-distance tweak. It is:

1. **Structural token models** the bijection-assuming hybrids miss — AbbrevAlign,
   RecurAlign, TokenRoleAlign.
2. **Calibration to probabilities** to kill magic thresholds — CalibFS, ChannelMix,
   BayesLenNorm.
3. **Performance-coupled scoring** where blocking and matching share one monotone
   function — BlockSimDual, AnytimeLev.

The single best bet by composite — and the one most aligned with goldenmatch's matchers —
is **AbbrevAlign**.

## Spotlight: AbbrevAlign (composite 84)

Soft-TFIDF, but the secondary token match is *asymmetric* under a small abbreviation
grammar, and a single token may align to a **span** of tokens (acronyms).

```text
score(A, B):
  for a in A:
    best[a] = max over (single b) sim_asym(a, b)            # token -> token
    for span S of B with |S| == |a|:                        # token -> span (acronym)
      best[a] = max(best[a], acronym_score(a, S))
  numer = sum_a w(a) * best[a];  denom = sum_a w(a)         # direction A -> B, IDF-weighted
  return mean( numer/denom (A->B), numer/denom (B->A) )     # symmetrized
```

It is a **strict generalization of Soft-TFIDF** (empty grammar recovers it), and the
one-to-many acronym alignment is genuinely absent from prior art. Falsifiable: F1 vs
Soft-TFIDF / Jaro-Winkler on abbreviation-heavy slices of Cora / DBLP-ACM / company-name
data. Reserved (non-fatal) concern: acronym grammar can over-fire (`IBM` vs `Indian Bank
Mumbai`) — mitigated by IDF weighting + calibration.
